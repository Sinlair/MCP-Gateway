MCP Gateway

令牌驱动的系统控制入口

这里不是传统账号密码后台，而是网关签发令牌之后的控制台入口。 用户拿到访问令牌后，系统会自动识别他可访问的环境、受管系统、工具范围和控制权限。

阶段一

网关签发令牌

令牌绑定 profile、roles、scopes、managedSystems 与 environment。

阶段二

控制台验证令牌

控制台只接受 Bearer Token，并基于它装载可见模块和可执行动作。

阶段三

进入监控与控制面

当前默认受管系统是 `big-market-71772-z`，同时保留网关运行态与工具治理视图。

访问控制台

输入访问令牌

支持 URL `?token=...` 自动带入，也会记住本地最近一次令牌。

本地演示

快速签发测试令牌

本地环境可直接签发演示令牌。生产环境应由 MCP Gateway 的系统配置流程签发正式令牌。
